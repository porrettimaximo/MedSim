## Nombre
HU-05 - Configurar backend LLM

## Objetivo
Como usuario, quiero configurar la URL y credenciales del backend LLM para usar distintos proveedores o modelos OpenAI-compatible.

## Contexto
El simulador desacopla el LLM: puede ser local (Ollama/vLLM) o remoto. La configuracion debe ser simple y verificable.

## Alcance
Incluye:
- Guardar `api_url` y `api_key` (si aplica) desde UI.
- Probar conectividad basica (`/models`).
- Sugerir URLs comunes (Ollama, puertos comunes, Gemini OpenAI-compatible, etc.).

No incluye:
- Administracion de usuarios o secretos (sandbox).
- Seleccion avanzada de modelos o parametros (se usa primer modelo disponible si no se especifica).

## Reglas de negocio
- El `base_url` se normaliza a una raiz OpenAI-compatible (`.../v1` o `.../v1beta/openai`).
- Se cachea el primer modelo por base_url para reducir llamadas.
- Dentro de un encounter se bloquea el modelo seleccionado para consistencia.

## Criterios de aceptacion
- Dado una URL valida Cuando guardo config Entonces el backend queda configurado.
- Dado una URL inaccesible Cuando verifico salud Entonces veo error claramente.

## Casos borde y errores
- URL vacia: 400.
- API key invalida: error al listar modelos.
- Backend responde sin modelos: error/estado no saludable.

## Datos y validaciones
Campos:
- `api_url` (string)
- `api_key` (string, opcional segun proveedor)

## Seguridad y roles
- Sin auth. Riesgo: la API key queda en localStorage del navegador.

## Dependencias e integraciones
- Cliente OpenAI-compatible (`openai.AsyncOpenAI`).
- Endpoints: `POST /api/config`, `GET /api/llm_health`, `GET /api/suggested_urls`.

## Flujo y UX
- Sidebar LLM: preset provider + input URL + API key + Save.
- Boton de salud/auto-deteccion (si aplica).

## Observabilidad
- Log de inicializacion de backend con base_url.

## Tests sugeridos
Unitarios:
- Normalizacion de base_url.
Integracion:
- `probe_base_url` a un servidor mock que responda `/models`.

## Datos de prueba
- Ollama: `http://127.0.0.1:11434`
- OpenAI-compatible: `http://localhost:8000/v1`

## Notas
- La UI puede bloquear cambios de proveedor durante una charla en curso para evitar inconsistencias.
