## Nombre
HU-01 - Gestionar pacientes base

## Objetivo
Como evaluador o administrador, quiero crear, editar, listar y eliminar pacientes para mantener actualizado el banco de casos clinicos del simulador.

## Contexto
La operacion diaria se hace sobre MongoDB y desde la UI de administracion.

## Alcance
Incluye:
- listado paginado y buscable de pacientes
- alta y edicion desde formulario web
- eliminacion de pacientes
- visualizacion de datos administrativos, historia, triage y caso real

No incluye:
- versionado de casos
- importacion masiva desde UI

## Reglas de negocio
- `PatientProfile.id` es el identificador estable del paciente.
- los pacientes se persisten en la coleccion `patients` de MongoDB.

## Criterios de aceptacion
- Dado un paciente existente, cuando abro `/frontend/patients`, entonces puedo verlo en la tabla.
- Dado un formulario valido, cuando guardo un paciente, entonces queda disponible para iniciar encounters.
- Dado un paciente existente, cuando lo elimino, entonces deja de aparecer en el ABM.

## Datos y validaciones
- entidad: `PatientProfile`
- campos visibles clave:
  - identificacion administrativa
  - motivo de consulta / triage
  - sintomas e historia
  - `true_case`

## Endpoints relacionados
- `GET /api/patients/`
- `GET /api/patients/{patient_id}`
- `POST /api/patients/`
- `DELETE /api/patients/{patient_id}`

## Flujo y UX
- el ABM vive en `/frontend/patients`
- la tabla permite buscar, paginar y editar
- el formulario concentra tanto datos visibles como el caso real del paciente

## Observabilidad
- errores de validacion o guardado se exponen en la UI

## Tests sugeridos
- crear paciente y verificar aparicion en listado
- editar un paciente y verificar persistencia
- eliminar un paciente y verificar ausencia en la tabla
