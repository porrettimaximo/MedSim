## Nombre
HU-07 - Configurar servicios y verificar el estado operativo

## Objetivo
Como mantenedor del entorno, quiero configurar MongoDB, LLM, STT y TTS y verificar su estado para poder operar el simulador de extremo a extremo.

## Contexto
La aplicacion depende de servicios externos configurados por `.env`. El backend expone un estado resumido para que el frontend y el operador sepan si el audio y el modelo estan listos.

## Alcance
Incluye:
- carga de variables de entorno
- conexion a MongoDB
- endpoint de estado de configuracion
- compatibilidad con proveedores OpenAI-compatible y variantes soportadas

No incluye:
- panel administrativo de credenciales
- rotacion automatica de secrets

## Reglas de negocio
- la app arranca conectando a MongoDB.
- al iniciar, el sistema asegura un paciente demo y un estudiante demo mediante bootstrap por upsert.
- `/api/config_state` informa si LLM, STT y TTS estan configurados.
- sin configuracion valida, la UI de audio se deshabilita o muestra error.

## Criterios de aceptacion
- Dado un `.env` valido, cuando levanto la app, entonces conecta a Mongo y expone la UI.
- Dado audio configurado, cuando abro el frontend, entonces el microfono puede habilitarse.
- Dado una configuracion invalida, cuando consulto `/api/config_state`, entonces veo el estado incompleto.

## Endpoints relacionados
- `GET /api/config_state`
- `GET /api/models`
- `GET /api/encounters_public`

## Flujo y UX
- el operador levanta FastAPI y verifica la configuracion
- el frontend consulta el estado para decidir si habilita audio

## Observabilidad
- el backend loguea conexion a Mongo y errores de proveedores
- el frontend muestra mensajes como `Audio no configurado`

## Tests sugeridos
- arranque con Mongo local
- consulta a `/api/config_state`
- prueba de chat texto
- prueba de mic, STT y TTS
