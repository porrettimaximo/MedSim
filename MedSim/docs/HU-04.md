## Nombre
HU-04 - Interactuar con el paciente por texto y audio

## Objetivo
Como estudiante, quiero conversar con el paciente simulado por texto o por voz para practicar la entrevista clinica y recibir respuestas en tiempo real.

## Contexto
La experiencia principal del producto ocurre en la vista del estudiante. El backend usa un LLM para responder como paciente, STT para transcribir audio y TTS para sintetizar la respuesta cuando esta habilitado.

## Alcance
Incluye:
- seleccion o adopcion de un encounter
- envio de mensajes por texto
- envio de audio por microfono
- recepcion de respuestas del paciente
- reproduccion de audios asociados a mensajes
- actualizacion en vivo por WebSocket

No incluye:
- evaluacion automatica del contenido clinico
- soporte offline

## Reglas de negocio
- el chat requiere `encounter_id`.
- los mensajes se guardan en `chat_history`.
- el audio se guarda en Mongo como asset y la UI lo reproduce via `audio_url`.
- si el encounter esta finalizado, la vista queda en solo lectura.

## Criterios de aceptacion
- Dado un encounter activo, cuando envio un texto, entonces el paciente responde y el mensaje queda guardado.
- Dado un encounter activo, cuando envio audio, entonces se transcribe, se guarda el audio y el paciente responde.
- Dado un mensaje con audio, cuando presiono reproducir, entonces el navegador consume `/api/audio/{audio_id}`.
- Dado un encounter finalizado, cuando intento enviar un mensaje, entonces la UI lo bloquea.

## Endpoints relacionados
- `POST /api/chat`
- `POST /api/audio_turn`
- `GET /api/encounters/{encounter_id}/history`
- `GET /api/audio/{audio_id}`
- `GET /api/config_state`
- `WS /ws/encounters/{encounter_id}`

## Flujo y UX
- la vista principal es `/frontend/student`
- `student_join` y `student_sessions` ayudan a entrar a una conversacion activa
- el panel lateral muestra la ficha clinica visible del paciente

## Observabilidad
- la UI informa estados como grabando, procesando, reproduciendo y errores
- el backend responde con 404 o 409 cuando el encounter no existe o esta cerrado

## Tests sugeridos
- chat por texto con respuesta del paciente
- chat por audio con STT y TTS
- reproduccion de audio historico
- bloqueo de sesion finalizada
