## Nombre
HU-02 - Gestionar estudiantes

## Objetivo
Como evaluador o administrador, quiero administrar estudiantes para poder asignarlos a sesiones simuladas y asociar sus evaluaciones.

## Contexto
Los estudiantes se usan al abrir una nueva sesion desde el dashboard del evaluador y sus datos quedan asociados al encounter y a la evaluacion SEGUE.

## Alcance
Incluye:
- listado de estudiantes
- alta y edicion
- eliminacion
- uso del alumno en el inicio de un encounter

No incluye:
- autenticacion real del alumno
- historiales academicos

## Reglas de negocio
- `StudentProfile.id` funciona como identificador estable.
- `student_identifier` se usa como legajo / DNI visible en UI y PDF.
- los estudiantes se persisten en la coleccion `students`.

## Criterios de aceptacion
- Dado un estudiante cargado, cuando entro a `/frontend/students`, entonces lo veo en la grilla.
- Dado un nuevo estudiante, cuando lo guardo, entonces aparece como opcion para iniciar una sesion.
- Dado un estudiante eliminado, cuando refresco el dashboard, entonces ya no aparece en el selector.

## Endpoints relacionados
- `GET /api/students/`
- `GET /api/students/{student_id}`
- `POST /api/students/`
- `DELETE /api/students/{student_id}`

## Flujo y UX
- el ABM vive en `/frontend/students`
- el dashboard del evaluador consume esa lista para iniciar sesiones

## Observabilidad
- la UI muestra mensajes de error y estado durante carga, guardado y eliminacion

## Tests sugeridos
- alta de estudiante y disponibilidad en dashboard
- edicion de nombre / identificador
- baja de estudiante
